# 极简联调清单（前后端对接当天照此走）

> **放置位置**：各复制一份到 `backend-repo/docs/` 和 `frontend-repo/docs/`。
> （docs/ 下的文件不会自动进 AI 上下文，保持代理上下文干净；要用时你或 AI 显式打开即可。）
> 性质：给**人**用的对接 runbook，不是给编码 AI 的逐阶段指令。

---

## 1. 启动顺序
1. 起 MySQL，执行 `drama_current_dump.sql` 建库 + 当前完整数据
2. 起后端 `mvn spring-boot:run`（:8080）→ 打开 `http://<后端IP>:8080/api/v1/doc.html` 确认接口都在
3. 前端改 `utils/config.js` 的 `BASE_URL`，`USE_MOCK` 置 `false`
4. 起前端预览 / 安装 APK

## 2. BASE_URL 怎么填（最常踩的坑）
| 场景 | BASE_URL |
|------|----------|
| 浏览器 H5 调试（同一台机器） | `http://localhost:8080/api/v1` |
| 真机 / 模拟器 / APK（同局域网） | `http://<后端电脑局域网IP>:8080/api/v1`　**绝不能用 localhost** |
| 生产 / 演示 | `https://<公网域名或IP>/api/v1` |
> 查局域网 IP：mac `ipconfig getifaddr en0`；win `ipconfig`（看 IPv4 地址）。

## 3. 对接冒烟测试（按依赖顺序逐个勾，全绿即闭环通）
- [ ] `GET /dramas` 列表出 2 部剧，前端列表页渲染正常
- [ ] 点击进播放页，`GET /episodes/101` 视频能播放
- [ ] `GET /episodes/101/highlights` 返回 3 条；播到 15s 弹"爽/名场面"**覆盖在视频上**
- [ ] 点"爽" → `POST /interactions` → 角标 +1，后端 DB 有明细
- [ ] 每 ~4s `GET /episodes/101/interaction-stats` → 飘屏 / 计数更新
- [ ] 播到 45s 弹分支 → 选"独自进入"(PREGEN) 播视频、"叫上伙伴"(ONDEMAND) 出文本
- [ ] 生成内容 `POST .../like` +1（重复点不加）；`POST .../comments` 后 `GET` 列表出现新评论

## 4. 字段一致性核对（分仓开发唯一的对接风险点）
对接前用两份 SPEC.md 的**第 4 节**逐接口比对：字段名、枚举值（`COOL/FUNNY/BRANCH`、`TEXT/VIDEO/IMAGE_SEQ`、`PREGEN/ONDEMAND`）、错误码。任一方改过字段，立即同步另一方。

## 5. 常见坑速查
| 现象 | 多半是 |
|------|--------|
| 真机白屏 / 请求失败 | BASE_URL 写了 localhost，改成局域网 IP |
| 跨域报错 | 后端 CORS 没放开 `/`，或 BASE_URL / 端口错 |
| 互动组件不显示（但有点击日志） | 用了普通 view，必须改 `cover-view` |
| 到点不弹组件 | `currentTime` 是秒，别和毫秒比；检查 startTime/endTime |
| 点赞能重复 +1 | 后端唯一约束没生效 / 前端没带 deviceId |
| 发评论 500 | content 为空或超 500，前端先校验 |
| 视频不播 | video_url 用了 http 但环境限制，换 https；或真机访问不到该地址 |
